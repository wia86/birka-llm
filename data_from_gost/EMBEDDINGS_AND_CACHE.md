# Эмбеддинги и кэш моделей

## Откуда берётся BAAI/bge-m3

**BAAI/bge-m3** — это одна конкретная модель с [Hugging Face](https://huggingface.co/BAAI/bge-m3). Репозиторий один, у него есть ветки (ревизии):

- **main** — основная стабильная версия (её по умолчанию качает `sentence-transformers` / `langchain-huggingface`).
- Другие ревизии (например `refs/pr/94` с ONNX) используются только при явном указании в коде.

В проекте задаётся только имя модели (`BAAI/bge-m3` или `intfloat/multilingual-e5-large`); ревизия не указывается — используется main. Версии в смысле "bge-m3 v1 / v2" у этой модели нет, это одна модель.

---

## Нужен ли PyTorch с CUDA

**Не обязательно.** Скрипты работают и на CPU:

- В `create_rag.py` проверяется `torch.cuda.is_available()`: если есть GPU — используется `cuda`, иначе `cpu`.
- На CPU создание эмбеддингов будет медленнее, но для небольших объёмов PDF это приемлемо.

Устанавливать PyTorch с CUDA имеет смысл, если:
- много PDF и нужна скорость;
- есть совместимая видеокарта NVIDIA и установлены драйверы + CUDA Toolkit.

Обычная установка: `pip install torch` (подставится вариант под вашу ОС; для CUDA смотрите [pytorch.org](https://pytorch.org)).

## Кэш Hugging Face и диск E:\lm

### `C:\Users\ADMIN\.cache\huggingface`

- **Назначение:** кэш моделей Hugging Face (в т.ч. эмбеддингов). Используется по умолчанию, если не заданы `HF_HOME` / `TRANSFORMERS_CACHE`.
- **Содержимое (типичное):** папка `hub` с подпапками вида `models--BAAI--bge-m3`, `models--intfloat--multilingual-e5-large` (скачанные веса и конфиги).
- **Актуальность:** если там уже есть `models--BAAI--bge-m3` и `models--intfloat--multilingual-e5-large`, повторно качать не нужно — проект их использует при создании RAG и в RAGAssistant.

### `E:\lm`

- **Содержимое (по проверке):** каталоги типа `DevQuasar`, `lmstudio-community`, `hub`, `mradermacher` и др. — похоже на каталог для LLM/моделей и, возможно, копии или симлинки кэша.
- **Использование в проекте:** по умолчанию проект **не** использует `E:\lm`. Чтобы хранить кэш Hugging Face на диске E:, в `.env` (или в переменных окружения) задайте, например:
  - `HF_HOME=E:\lm`
  - или `TRANSFORMERS_CACHE=E:\lm\hub`
  После этого новые загрузки моделей пойдут в указанный каталог; уже скачанные в `C:\Users\ADMIN\.cache\huggingface` продолжат оттуда использоваться, пока не заданы эти переменные.

## Как это используется в проекте

- **create_rag.py** — при запуске читает из окружения (в т.ч. из `.env`) `RAG_PERSIST_DIR`, `RAG_SOURCE_PATHS`, `RAG_MODEL_NAME`; загружает модель эмбеддингов (из кэша HF или по сети), строит векторную базу Chroma и сохраняет её в `RAG_PERSIST_DIR`.
- **RAGAssistant** (`assistant.py`) — при инициализации загружает ту же модель эмбеддингов (по имени из профиля) и Chroma из `persist_directory`; использует эмбеддинги для поиска релевантных чанков и для запросов к LLM.
- **load_llm.py** — фабрика эмбеддингов и тест модели; при запуске как скрипт тоже подхватывает `.env` (например, для `HF_HOME` / `RAG_*`).

Переменные окружения подхватываются из файла `.env` в корне проекта (birka-llm), если установлен `python-dotenv` и вызов `load_dotenv()` выполняется в точке входа (как в перечисленных скриптах).
