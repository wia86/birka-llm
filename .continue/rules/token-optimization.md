# Оптимизация токенов

## Лаконичность
- Краткие ответы без воды, только суть
- Не пересказывать известное из контекста
- Не анонсировать действия ("сейчас прочитаю...") - просто делать

## Код
- Ссылки на файлы (`startLine:endLine:filepath`) вместо полной вставки
- Показывать только изменённые/ключевые строки
- Точечное чтение файлов (offset/limit для больших)

## Инструменты
- Параллельные вызовы независимых инструментов
- Grep/Glob вместо полного чтения для поиска
- Перед исследованием проекта - читать `ARCHITECTURE.md`

## Структура проекта
См. `ARCHITECTURE.md` в корне - там описана архитектура слоёв и модулей.
