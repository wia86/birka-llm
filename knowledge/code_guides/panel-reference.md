# Panel Reference

Справочник пользовательских панелей Birka для support-RAG.

## Как читать

Для каждой панели указано:
- где используется;
- что вводит пользователь;
- что проверять саппорту;
- что ломается чаще всего.

## Источник и общие панели

### Выбор моделей
- Модули: `calc`, `edit`, `static`, `tkz`
- Назначение: выбрать файл, папку или папку со вложенными папками
- Проверять:
  - выбран ли активный путь;
  - соответствует ли тип источника сценарию;
  - не исключены ли нужные файлы
- Частые проблемы:
  - пустой источник;
  - старый путь из другого рабочего места;
  - пользователь выбрал не тот режим источника

### Сохранить как
- Модули: `calc`, `edit`, `static`, `tkz`
- Назначение: задать путь сохранения результата
- Проверять:
  - заполнено ли активное поле сохранения;
  - соответствует ли путь выбранному типу источника
- Частые проблемы:
  - путь не указан;
  - пользователь путает входной путь и путь сохранения

## Панели исходных данных

### Контролируемые элементы
- Модули: `calc`, `static`
- Назначение: задать контролируемые узлы, ветви и сечения
- Основные поля:
  - использовать поле `control`
  - автоматический выбор по выборке
  - таблица допустимых значений
- Проверять:
  - выбран хотя бы один способ задания;
  - выборка не пуста, если включен автоматический режим
- Частые проблемы:
  - не выбран ни один способ;
  - выборка не соответствует ожиданию пользователя

### Отключаемые элементы: комбинации
- Модули: `calc`, `static`
- Назначение: настроить типы СРС и выбор отключаемых элементов
- Основные поля:
  - чекбоксы типов СРС;
  - фильтр сочетаний;
  - выбор по `disable` или по выборке
- Проверять:
  - включен хотя бы один тип СРС;
  - отключаемые элементы заданы
- Частые проблемы:
  - нет выбранных сочетаний;
  - панель включена, но фактически пустая

### Отключаемые элементы: excel
- Модули: `calc`, `static`
- Назначение: загрузить отключаемые элементы из Excel
- Проверять:
  - указан ли файл;
  - подходит ли файл пользователю по сценарию
- Частые проблемы:
  - отсутствует файл;
  - пользователь ожидает другой тип входных данных

### Загрузка файла формата RastrWin3
- Модули: `calc`, `edit`, `static`, `tkz`
- Назначение: загрузить поддерживаемый файл RastrWin3
- Проверять:
  - правильное расширение;
  - пользователь не пытается загрузить `.rg2` или `.rst` в этот сценарий
- Частые проблемы:
  - выбран неподходящий формат файла

## Панели результатов `calc`

### Сводные таблицы с нарушениями
- Назначение: сформировать сводные нарушения
- Проверять:
  - панель действительно включена;
  - пользователь ожидает именно табличный результат
- Частые проблемы:
  - пользователь не выбрал входные данные для этого типа результата

### Выбор оборудования
- Назначение: результаты выбора оборудования
- Проверять:
  - связку с контролируемыми элементами;
  - корректность ожиданий пользователя по результату

### Расчеты в табличном виде
- Назначение: таблицы по результатам расчета
- Проверять:
  - отмечены ли конкретные таблицы;
  - не забыто ли название приложения/таблиц
- Частые проблемы:
  - пользователь включил панель, но не отметил ни одной таблицы

### Расчеты в графическом виде
- Назначение: сформировать графические материалы и связанные форматы
- Проверять:
  - нужен ли RG2;
  - нужен ли docx/PDF;
  - корректен ли шаблон имени;
  - включены ли нужные флаги СРС
- Частые проблемы:
  - не настроен внешний компонент для docx;
  - пользователь ждет рисунки, но не включил нужный формат

### Пояснительная записка
- Назначение: сформировать текстовый документ по результатам
- Проверять:
  - задано ли название проекта;
  - выбран ли корректный формат описания;
  - нужны ли рисунки в составе результата
- Частые проблемы:
  - пользователь не понимает различие между форматами описания

### БРМ
- Назначение: сформировать материалы по балансу реактивной мощности
- Проверять:
  - заполнена ли выборка;
  - заданы ли названия таблиц и раздела
- Частые проблемы:
  - пользователь пропускает выборку;
  - ожидает результат на всех данных без явного указания

### БМ
- Назначение: сценарий по коэффициентам и температурным данным
- Проверять:
  - используется ли `из СИПР` или `из РМ`;
  - доступны ли связанные данные;
  - корректен ли выбор ОЭС или области
- Частые проблемы:
  - внешние данные недоступны;
  - пользователь не выбрал нужный источник данных

## Панели `tkz`

### Добавить оборудование
- Назначение: подготовить оборудование для расчета
- Проверять:
  - панель присутствует;
  - она расположена до панели расчета

### Расчет ТКЗ
- Назначение: основной расчет ТКЗ
- Проверять:
  - порядок панелей;
  - выборка узлов, если используется
- Частые проблемы:
  - нарушение порядка панелей

## Панели `edit` и `other`

### Импорт из РМ
- Назначение: импортировать данные из файла РМ
- Проверять:
  - указан файл;
  - выбрана таблица;
  - корректен режим слияния
- Частые проблемы:
  - пустой файл;
  - неподходящий режим импорта

### Выполнить задание из файла
- Модуль: `other`
- Назначение: исполнить уже сохраненное задание
- Проверять:
  - соответствует ли файл модулю;
  - нужна ли загрузка настроек из файла
- Частые проблемы:
  - пользователь пытается выполнить неподходящий файл

### Выполнить задания из папки
- Модуль: `other`
- Назначение: пакетный запуск по папке
- Проверять:
  - не пустая ли папка;
  - не ломается ли один из файлов в наборе

### Создать PDF / EXCEL в WORD (PDF)
- Модуль: `other`
- Назначение: конвертация офисных файлов
- Проверять:
  - входной файл;
  - настройки формата;
  - ожидаемый результат пользователя

### Создать графику для rg2/rst
- Модуль: `other`
- Назначение: графическая обработка файлов
- Проверять:
  - формат входного файла;
  - параметры вывода

### Граф из JSON в .grf
- Модуль: `other`
- Назначение: конвертация JSON в графический формат
- Проверять:
  - корректность входного JSON;
  - что пользователь выбрал именно этот сценарий
