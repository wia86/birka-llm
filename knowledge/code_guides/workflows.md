# Workflows

Пошаговые сценарии для техподдержки. Все шаги описаны в терминах интерфейса Birka и подходят для ответов пользователю.

## Создать новое задание и запустить расчет

### Когда использовать
Пользователь хочет выполнить новый расчет, а не открыть уже готовый файл задания.

### Шаги
1. Открыть нужный модуль: `Расчет установившихся режимов`, `Изменение и анализ РМ`, `Расчет статической устойчивости`, `Расчет ТКЗ` или `Прочее`.
2. В верхнем блоке `Выбор моделей` указать источник:
   - `Файл`, если расчет идет по одной модели;
   - `Папка`, если используется набор файлов;
   - `Папка со вложенными папками`, если расчет нужно запускать пакетно.
3. Добавить нужные панели заданий в рабочую область.
4. Заполнить обязательные поля. Если поле не заполнено, оно обычно подсвечивается.
5. При необходимости сохранить задание в файл.
6. Запустить выполнение.

### Как понять, что все прошло успешно
- показывается сообщение об успешном завершении;
- появляется папка результатов или набор выходных файлов;
- в результате есть сообщения категорий `Информация` или `Предупреждение`, но нет критического сбоя.

### Если запуск не прошел
- Проверить [errors-and-fixes.md](errors-and-fixes.md).
- Если появился traceback, перейти к [traceback-triage.md](traceback-triage.md).

## Открыть существующий файл задания

### Когда использовать
Пользователь прислал файл `.calc`, `.edit`, `.static`, `.tkz` или `.other`, и нужно повторить его сценарий.

### Шаги
1. Открыть модуль, соответствующий расширению файла.
2. Выбрать команду загрузки задания или перетащить файл задания на главный экран.
3. Убедиться, что источник данных и панели восстановились корректно.
4. Проверить, не появились ли пустые пути, отключенные панели или устаревшие названия панелей.
5. При необходимости обновить путь к входным данным и запустить выполнение повторно.

### Что проверить после загрузки
- модуль совпадает с расширением файла;
- в задании есть источник РМ;
- включены нужные панели;
- обязательные поля не пустые;
- пользователь понимает, какой результат должен быть получен.

### Если файл не открывается или ведет себя странно
- Использовать [task-file-triage.md](task-file-triage.md).
- Проверить кодировку файла и структуру YAML.

## Настроить контролируемые элементы

### Когда использовать
Для модулей `calc` и `static`, если пользователь хочет получить результаты по контролируемым элементам.

### Шаги
1. Добавить панель `Контролируемые элементы`.
2. Выбрать один или оба способа задания:
   - отмеченные элементы в поле `control`;
   - автоматический выбор по выборке узлов.
3. Если нужен отдельный контроль допустимых значений, включить соответствующий флаг.
4. Проверить, что хотя бы один способ выбора действительно задан.

### Типовой результат
- расчет использует выбранные узлы, ветви и сечения;
- становятся доступны связанные панели результатов.

### Если пользователь получил предупреждение
- Обычно это значит, что контролируемые элементы не выбраны или не совпадают с выбранными результатами.
- См. раздел `Не выбраны контролируемые элементы` в [errors-and-fixes.md](errors-and-fixes.md).

## Настроить отключаемые элементы и СРС

### Когда использовать
В `calc` и `static`, если результат зависит от сочетаний отключений и температурных условий.

### Шаги
1. Добавить панель `Отключаемые элементы: комбинации`.
2. Выбрать нужные типы СРС.
3. При необходимости включить фильтр сочетаний с 2 и 3 ветвями и задать процент взаимного влияния.
4. Выбрать отключаемые элементы:
   - по полю `disable`;
   - или по выборке.
5. Убедиться, что панель не отключена.

### Частые проблемы
- не выбран ни один тип СРС;
- не выбраны отключаемые элементы;
- пользователь ожидает результаты, но панель фактически отключена.

## Сформировать табличные и графические результаты

### Когда использовать
В `calc`, если пользователь хочет таблицы, рисунки, пояснительную записку, БРМ или БМ.

### Шаги
1. Добавить одну или несколько панелей результатов.
2. Для `Расчеты в табличном виде` отметить нужные таблицы.
3. Для `Расчеты в графическом виде` выбрать, сохранять ли все СРС, только отклонения, RG2, docx, PDF и данные для рисунков.
4. Для `Пояснительной записки` указать название проекта, раздела и формат описания.
5. Для `БРМ` и `БМ` проверить специфические поля выбора.

### Что важно для саппорта
- одна и та же панель результата в разделе результатов не должна быть включена дважды;
- графические сценарии зависят от корректных входных данных и дополнительных внешних компонентов;
- часть результатов требует сначала корректно задать контролируемые элементы.

## Выполнить сценарий ТКЗ

### Когда использовать
Пользователь работает в модуле расчета ТКЗ.

### Шаги
1. Указать источник РМ.
2. При необходимости добавить панель исправления номеров узлов.
3. Добавить панель оборудования.
4. Добавить панель `Расчет ТКЗ`.
5. Проверить порядок панелей: оборудование должно быть перед расчетом.
6. Запустить выполнение.

### Что проверять в первую очередь
- корректность входного файла;
- наличие панели оборудования;
- порядок включенных панелей;
- заполненность выборки узлов, если она используется.

## Выполнить пакетный сценарий из файла задания

### Когда использовать
Пользователь прислал готовый файл задания и ожидает, что программа выполнит его без ручной настройки каждой панели.

### Шаги
1. Определить модуль по расширению файла.
2. Загрузить файл в соответствующий модуль или использовать сценарий в модуле `Прочее`.
3. Проверить:
   - вид источника данных;
   - пути к файлам;
   - список панелей;
   - включенность панелей;
   - обязательные поля.
4. Если пути устарели, попросить пользователя переуказать источник данных.
5. Повторно запустить выполнение.

### Если пользователь прислал только файл без traceback
- сначала разобрать файл по [task-file-triage.md](task-file-triage.md);
- если по структуре файла проблема неясна, попросить traceback или лог только после первичной проверки.

## Разобрать ошибку по traceback

### Когда использовать
Пользователь прислал текст ошибки или скрин с деталями исключения.

### Шаги
1. Выделить тип исключения, сообщение и последние 5-10 кадров.
2. Определить стадию сценария:
   - загрузка задания;
   - проверка полей;
   - запуск расчета;
   - выполнение дочернего процесса;
   - обработка результатов.
3. Перевести ошибку в пользовательский язык: какая операция не удалась.
4. Проверить, какие данные и панели относятся к этому шагу.
5. Дать безопасные действия пользователю.
6. Если ошибка повторяется после корректировки данных, подготовить эскалацию.
