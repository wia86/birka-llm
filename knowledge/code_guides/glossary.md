# Glossary

Краткие термины и расшифровки для ответов техподдержки по Birka.

- **Задание** — файл сценария работы программы, в котором сохранены источник данных, панели и параметры.
- **Файл задания** — YAML-файл с расширением модуля, например `.calc`, `.edit`, `.static`, `.tkz`, `.other`.
- **Модуль** — крупный раздел программы, например расчет режимов, ТКЗ или изменение РМ.
- **Панель задания** — блок в рабочей области, отвечающий за один шаг сценария или один тип результата.
- **Источник РМ** — файл, папка или папка со вложенными папками, из которых программа берет исходные данные.
- **РМ** — расчетная модель, основной входной файл или набор файлов для сценариев программы.
- **Контролируемые элементы** — узлы, ветви или сечения, по которым программа проверяет результат и формирует часть выходных данных.
- **Отключаемые элементы** — элементы схемы, отключение которых участвует в расчете сочетаний и сценариев.
- **СРС** — сценарии/состояния расчетной схемы, используемые в расчете сочетаний и результатов.
- **Выборка** — условие или фильтр, по которому программа ограничивает набор узлов, элементов или записей.
- **Результаты расчетов** — выходные панели и файлы: таблицы, рисунки, пояснительные записки и другие формы результата.
- **Пояснительная записка** — документ с текстовым описанием результатов расчета.
- **БРМ** — сценарий формирования материалов по балансу реактивной мощности.
- **БМ** — сценарий, связанный с коэффициентами зависимости потребления от температуры и сопутствующими данными.
- **ТКЗ** — расчет токов короткого замыкания.
- **Пакетная обработка** — запуск сценария сразу для группы файлов или вложенных папок.
- **Обязательное поле** — поле панели, которое должно быть заполнено для корректного запуска сценария.
- **Отключенная панель** — панель, которая сохранена в задании, но не участвует в выполнении.
- **Traceback** — текст стека исключения Python, который показывает, на каком шаге произошла ошибка.
- **Triage** — первичный разбор проблемы с целью быстро определить причину, шаг сценария и безопасные действия.
- **Curated knowledge** — подготовленный безопасный набор markdown-материалов для RAG вместо прямой индексации всего кода.
- **Allowlist** — перечень файлов и директорий, которые разрешено индексировать.
- **Denylist** — перечень признаков чувствительных данных, которые нельзя оставлять в материалах для индексации.
- **RAG** — Retrieval-Augmented Generation, схема ответа LLM с опорой на подготовленную базу знаний.
