# Features Overview

Карта пользовательских возможностей Birka для RAG техподдержки. Документ описывает функции в терминах интерфейса и сценариев работы, без низкоуровневых деталей реализации.

## Главные разделы программы

| Модуль | Назначение | Типовые входные данные | Основной результат |
|---|---|---|---|
| `calc` | Расчет установившихся режимов | Расчетные модели РМ, панели выбора контролируемых и отключаемых элементов, панели результатов | Таблицы, рисунки, пояснительная записка, диагностические сообщения |
| `edit` | Изменение и анализ РМ | Файлы РМ и панели редактирования/анализа | Измененные файлы, выборки, сравнение РМ, выгрузки |
| `static` | Расчет статической устойчивости | РМ, параметры расчета, панели результатов | Таблицы и связанные результаты по статике |
| `tkz` | Расчет токов короткого замыкания | Файлы РМ, панели оборудования и параметров ТКЗ | Таблицы ТКЗ, подготовка данных, узлы и присоединения |
| `other` | Прочие утилиты и конвертация | Файлы заданий, Excel, JSON, RG2/RST, базы данных | Конвертация, экспорт, пакетная обработка, служебные документы |

## Общая модель работы пользователя

1. Пользователь выбирает модуль на главном экране.
2. Указывает источник данных: файл РМ, папку или папку со вложенными папками.
3. Добавляет панели заданий по нужному сценарию.
4. Заполняет обязательные поля на панелях.
5. Сохраняет задание в файл или сразу запускает выполнение.
6. Получает результат, диагностические сообщения или traceback.

## Единицы интерфейса

### Задание
Файл задания в формате YAML с расширением по модулю: `.calc`, `.edit`, `.static`, `.tkz`, `.other`.

### Панель задания
Отдельный блок в рабочей области с параметрами одного шага. Панели можно включать, отключать, дублировать и удалять.

### Источник РМ
Верхний блок выбора входных моделей:
- `Файл`
- `Папка`
- `Папка со вложенными папками`

Для вложенных папок поддерживаются исключения по файлам и каталогам.

### Категории панелей
- `Редактировать`
- `Анализ данных`
- `Исходные данные`
- `Результаты расчетов`
- `Конвертация форматов`
- `Прочее`

## Возможности по модулям

### `calc` — расчет установившихся режимов

#### Исходные данные
- `Контролируемые элементы`: выбор по полю `control` или по выборке узлов.
- `Отключаемые элементы: комбинации`: задание типов СРС, фильтра сочетаний и выборки отключаемых элементов.
- `Отключаемые элементы: excel`: импорт отключаемых элементов из таблицы.
- `Загрузка файла формата RastrWin3`: загрузка файлов форматов `sch`, `grf`, `trn`, `anc`, `ut2`, `kpr`.

#### Результаты расчетов
- `Сводные таблицы с нарушениями`
- `Выбор оборудования`
- `Расчеты в табличном виде`
- `Расчеты в графическом виде`
- `Пояснительная записка`
- `БРМ`
- `БМ`

#### Что важно для саппорта
- Для расчета обычно нужен источник РМ и минимум одна включенная панель результата.
- Часть результатов требует контролируемые элементы.
- Часть сценариев пропускает отдельные проверки, если выбран определенный тип результата.

### `edit` — изменение и анализ РМ

#### Основные сценарии
- `Импорт из РМ`
- `Изменение РМ (в текстовом формате)`
- `Изменение РМ (в формате excel)`
- `Сравнение РМ`
- `Вывод данных из таблицы РМ`
- `Фильтр РМ`
- `Рисунки РМ`
- `Распознать граф РМ`

#### Типовой результат
- Измененный набор файлов.
- Табличная выгрузка.
- Аналитическая выборка.
- Подготовка рисунков и сопутствующих файлов.

### `static` — расчет статической устойчивости

#### Основные сценарии
- Использует общие механизмы выбора источника РМ.
- Позволяет задавать параметры расчета.
- Поддерживает панели результатов, близкие к режимным расчетам.

#### Что важно для саппорта
- Проверки похожи на `calc`: источник данных, корректность панелей, заполненность обязательных полей.

### `tkz` — расчет ТКЗ

#### Основные сценарии
- `Исправить номера узлов в РМ`
- `Добавить оборудование`
- `Оборудование: добавить ссылки на РМ`
- `Расчет ТКЗ`

#### Особенности
- Для части сценариев важен порядок панелей.
- Панель оборудования должна идти раньше панели расчета ТКЗ.
- Результаты включают выборку узлов и дополнительные флаги вывода.

### `other` — прочие инструменты

#### Конвертация и пакетная обработка
- `Выполнить задание из файла`
- `Выполнить задания из папки`
- `Создать PDF`
- `EXCEL в WORD (PDF)`
- `Создать графику для rg2/rst`
- `Граф из JSON в .grf`
- `Создать карту схему в .dxf`
- `Пояснительная записка УР`

#### Что важно для саппорта
- Этот модуль часто используется как сервисный: запуск по уже сохраненным заданиям, конвертация и пакетные операции.
- Для разбора проблем важно сначала определить тип входного файла и ожидаемый тип результата.

## Файлы заданий и запуск

### Что хранится в файле задания
- версия формата/приложения;
- тип модуля;
- описание источника данных;
- список панелей и их параметры;
- безопасная часть пользовательских настроек.

### Что не следует использовать в ответе пользователю как есть
- абсолютные пути к файлам;
- служебные внутренние настройки;
- полный traceback без маскировки;
- данные лицензии, логов и инфраструктуры.

## Диагностика и сообщения

### Источники пользовательской диагностики
- встроенные проверки перед запуском;
- подсветка незаполненных обязательных полей;
- диалоги предупреждений и подтверждений;
- итоговые сообщения расчета по категориям: ошибка, предупреждение, информация;
- traceback при сбое.
